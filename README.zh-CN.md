# Elite Oracle 中文说明

**Elite Oracle（精英神谕）** 是一个可复用的 Codex 技能与 AI 同事画像。它从世界顶级知识机构的公开输出中，提炼出一套可执行的战略分析操作系统。

这些机构类型包括：战略咨询公司、顶级风险投资机构、研究分析机构、投资银行、资产管理机构、宏观研究机构，以及管理学和经济学研究机构。本项目不会模仿任何单一机构，也不声称拥有任何专有方法或内部资料。它提炼的是公开内容中可复用的分析习惯、问题结构和输出标准。

## 适用场景

- 市场研究
- 竞争格局分析
- AI 与技术趋势分析
- 创业方向判断
- 风险投资 thesis
- 投资备忘录
- 管理层 briefing
- 咨询式 deck 结构
- 行业结构分析
- 并购、资本市场和宏观环境分析

## 核心问题

Elite Oracle 会把复杂问题压缩成七个决策问题：

1. 什么正在变化？
2. 为什么是现在？
3. 谁会赢，谁会输？
4. 支撑判断的证据是什么？
5. 隐藏的约束是什么？
6. 下一步应该做什么？
7. 出现什么信号时，我们应该改变判断？

## 提炼出的思维模式

- **McKinsey / MGI 型:** 把高度不确定的问题拆解成高层管理者可以决策的议题。
- **BCG / Henderson Institute 型:** 用竞争优势、战略假设、组织能力和情景推演理解未来。
- **Bain 型:** 强调客户经济、利润池、执行 KPI、私募股权式的实践纪律。
- **Big Four / Accenture 型:** 关注大型组织转型、治理、风险、监管和落地可行性。
- **a16z / Sequoia / YC 型:** 关注技术浪潮、创业 wedge、市场创造、分发路径和创始人真相。
- **Gartner / Forrester / IDC 型:** 建立分类体系、成熟度曲线、采用时间表和供应商比较。
- **Goldman / JPMorgan / Morgan Stanley 型:** 连接宏观环境、资本流动、估值、并购和盈利驱动因素。

## 12 个分析镜头

1. 市场结构：规模、增长、集中度、利润池、瓶颈。
2. 客户现实：痛点、预算拥有者、切换成本、采用摩擦。
3. 技术成熟度：能力曲线、成本曲线、可靠性、互操作性。
4. 商业模式：单位经济、变现方式、利润率、收入持续性。
5. 竞争优势：数据、渠道、人才、品牌、监管、工作流锁定。
6. 资本市场：融资环境、估值压力、并购路径、退出窗口。
7. 运营模式：组织设计、流程变化、激励、治理、执行节奏。
8. 风险与监管：合规、安全、声誉、地缘政治。
9. 时机：成熟度曲线、炒作周期、S 曲线、触发事件。
10. 情景推演：基准情景、上行情景、下行情景、非连续变化。
11. 指标：领先指标、滞后指标、阈值、失败信号。
12. 叙事：一句话说明为什么这件事现在重要。

## 使用方法

在 Codex 中使用时，把 `skill/SKILL.md` 放到：

```text
~/.codex/skills/elite-oracle/SKILL.md
```

然后这样调用：

```text
Use Elite Oracle to analyze whether agentic AI will reshape enterprise SaaS.
```

也可以把 `colleague/work.md` 和 `colleague/persona.md` 提供给任意 AI 模型，让它按照同样的分析风格工作。

## 输出模式

- **Executive Brief:** 面向管理层的决策简报。
- **VC Thesis:** 面向创业和风险投资的市场假设。
- **Consulting Deck Narrative:** 咨询式汇报结构和 storyline。
- **Analyst Landscape:** 市场、技术、品类或供应商格局。
- **Investment / Macro Memo:** 投资、并购、估值、资本市场和宏观分析。

## 注意

本项目仅基于公开输出中可观察的通用分析模式，不包含任何机构的机密资料、内部方法或专有研究。
