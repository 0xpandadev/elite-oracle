# 世界トップ知的集団のアウトプット・リバースエンジニアリング用リスト

調査日: 2026-04-20  
範囲: 公開Webで確認できる公式・準公式アウトプット中心。Gartner/Forrester/IDCなど一部は概要ページのみ公開で、全文は有料の場合あり。

## 使い方

1. まず「定点観測ハブ」をRSS/ブックマーク化する。
2. 次に「フラッグシップ年次レポート」を10本選び、構成を分解する。
3. 1本ごとに、問い、仮説、データ、セグメント、フレームワーク、示唆、推奨アクションを抜き出す。
4. 最後に自分用テンプレートへ変換する。例: `市場構造 -> 非連続変化 -> 勝者の条件 -> 具体施策 -> リスク`。

## 先に盗むべき型

- McKinsey型: 巨大データを「経営者が意思決定できる粒度」に分解する。
- BCG型: 戦略論を「勝ち筋、組織能力、実装ロードマップ」に落とす。
- Bain型: PE/顧客/オペレーション視点で、実務に近いKPIに寄せる。
- a16z/Sequoia型: 技術トレンドを「市場構造の変化」と「次の巨大企業の条件」に接続する。
- Gartner/Forrester/IDC型: 分類軸、成熟度、導入時期、ベンダー比較を標準化する。
- 投資銀行型: マクロ、資本市場、M&A、セクター見通しを「資金の流れ」と「バリュエーション」に接続する。

## 逆算テンプレート

各レポートを読むときは、この7項目に分解する。

1. 問い: この資料は何を意思決定させたいのか。
2. 単位: 国、産業、企業、顧客、機能、技術、資本市場のどれを分析単位にしているか。
3. 切り口: 成熟度、成長率、収益性、採用率、資本流入、リスク、競争優位のどれで割っているか。
4. データ: 独自調査、公開統計、取引データ、アンケート、専門家インタビュー、モデル推計のどれか。
5. 図表: 2x2、成熟曲線、ウォーターフォール、ヒートマップ、ランキング、時系列、バリューチェーンのどれか。
6. 示唆: 「だから何か」を1文で言うと何か。
7. 行動: 経営者、投資家、創業者、政策担当者が明日やることは何か。
